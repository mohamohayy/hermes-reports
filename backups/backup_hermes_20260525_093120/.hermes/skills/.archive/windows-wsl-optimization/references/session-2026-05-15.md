### Windows-side commands (write as .ps1, run with -File)

```powershell
# Startup items
Get-CimInstance Win32_StartupCommand | Select-Object Name, Command | Format-List

# Top 10 memory processes
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 10 | ForEach-Object {
    $mem = [math]::Round($_.WorkingSet64 / 1MB, 1)
    Write-Host "$($_.Name) - ${mem}MB"
}

# Disk overview
Get-PSDrive -PSProvider FileSystem | Where-Object { $_.Used -gt 0 } | ForEach-Object {
    $used = [math]::Round($_.Used / 1GB, 1)
    $free = [math]::Round($_.Free / 1GB, 1)
    $total = $used + $free
    Write-Host "$($_.Name): ${used}GB used / ${total}GB total (${free}GB free)"
}
```

## Pitfalls hit

1. **PowerShell inline `$_` mangled by bash** — `$_.WorkingSet64` became garbled path. Fixed by writing `.ps1` file via `write_file` (which produces clean UTF-8) then running with `-File`.
2. **PowerShell output garbled (GBK/UTF-8 mismatch)** — Chinese column headers displayed as 乱码. Fixed by rewriting `.ps1` in pure ASCII/English.
3. **Non-sudo journald cleanup partial** — can vacuum `/run/log/journal` but not `/var/log/journal` without password. Fallback: write script to Desktop, user runs manually.
4. **venv had no pip** — `python3 -m ensurepip --upgrade` then `python3 -m pip install ...`.
